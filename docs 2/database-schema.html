Finvision – Database Schema

Database: MongoDB (NoSQL document store)  |  Driver: Spring Data MongoDB  |  Collections: 4

Collections: users bills budgets scanned_receipts

Finvision uses MongoDB as its sole database. Because MongoDB is a document-oriented NoSQL database, there is no SQL DDL schema. The structure of each collection is enforced at the application layer by the Java model classes annotated with @Document (Spring Data MongoDB). Each document in a collection corresponds to an instance of one of these classes.

Collection: users

Collection: budgets

Budget

One document per user per calendar month. Stores planned income and categorised expense amounts for that month.

Java class: com.finvision.model.Budget

Field Type Constraints Description
_id String (ObjectId) required indexed MongoDB auto-generated primary key.

monthlyIncome double required Primary monthly income (salary, wages, etc.).

rent double optional Budgeted rent or mortgage expense. Defaults to 0.

groceries double optional Budgeted groceries expense. Defaults to 0.
subscriptions double optional Budgeted subscription services expense. Defaults to 0.
variableTitle Array<String> optional User-defined variable expense category names (parallel array with variableAmount).
variableAmount Array<Double> optional Dollar amounts for each user-defined variable expense (parallel array with variableTitle).

Collection: scanned_receipts

ScannedReceipt

One document per manually entered or scanned receipt. Records actual spending transactions used in Budget vs. Actual comparisons.

Java class: com.finvision.model.ScannedReceipt

Index: Compound index { username: 1, yearMonth: 1 } named username_yearmonth_idx — enables fast per-month receipt queries.

Field Type Constraints Description
_id String (ObjectId) required indexed MongoDB auto-generated primary key.
username String required indexed Owner of this receipt (foreign key to users.username).
merchantName String optional Name of the merchant or payee. Defaults to "Unknown" if not provided.
amount double required Dollar amount of the transaction.
category String required Spending category. One of: Groceries, Dining, Shopping, Transport, Healthcare, Entertainment, Rent, Utilities, Insurance, Subscriptions, Other.
dateStr String (ISO date) required Date of the transaction in YYYY-MM-DD format.
yearMonth String required indexed First 7 characters of dateStr (e.g. "2026-04"). Used for fast month-filtered queries.
notes String optional Free-text notes about the transaction.
imageData String optional Base64-encoded receipt image thumbnail (e.g. data:image/jpeg;base64,…).
